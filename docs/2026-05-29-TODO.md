### OpenClaw Dockerfile cleanup + bake-in toolbox symlink
Two things to bundle next time the Dockerfile is touched:

1. **Redundant npm globals.** `docker/openclaw/Dockerfile` re-installs
   `pi-coding-agent`, `pi-acp`, `opencode-ai` even though the devbox base
   already provides them. Harmless wart.

2. **`~/.agents → /projects/toolbox` symlink.** Openclaw discovers personal
   skills by scanning `~/.agents/skills/` (`source = agents-skills-personal`).
   The dotfiles repo no longer chezmoi-externals into `~/.agents` (replaced
   with the `~/toolbox -> /projects/toolbox` symlink_template), so after the
   next rebuild `~/.agents` will not exist inside openclaw → `agents-skills-personal`
   sees zero skills.

   Manual one-off fix is `docker exec openclaw ln -s /projects/toolbox /home/ankit/.agents`,
   but it's lost on rebuild. To bake it in: either add the symlink in the
   `Dockerfile` (`RUN ln -s /projects/toolbox /home/ankit/.agents`) — but
   `/projects` isn't present at build time, so the symlink will be dangling
   until the bind-mount activates at runtime (still works as a symlink
   target, just looks broken to `ls` mid-build) — or do it in
   `docker/openclaw/entrypoint.sh` (preferred):
   ```sh
   [ -e /home/ankit/.agents ] || ln -s /projects/toolbox /home/ankit/.agents
   ```

**Trigger:** next time that Dockerfile is touched for any other reason.
